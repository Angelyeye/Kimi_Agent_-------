# 《春节模拟器》任务分配建议

## 概述

本文档提供各Agent的任务分配建议，便于并行开发和协作。

---

## Agent角色定义

| Agent代号 | 职责领域 | 主要技能 |
|----------|---------|---------|
| Agent-策划 | 游戏策划、文案 | 游戏设计、创意写作 |
| Agent-美术 | UI设计、美术资源 | 视觉设计、图像处理 |
| Agent-数据 | JSON数据配置 | 数据结构、配置管理 |
| Agent-前端 | HTML/CSS/JS开发 | 前端开发、交互实现 |
| Agent-测试 | 测试计划与执行 | 测试设计、Bug跟踪 |
| PM(本Agent) | 项目管理、协调 | 进度跟踪、风险管理 |

### 关键路径

```
核心玩法设计 → 角色设定 → 故事线文案 → JSON配置 → 前端开发 → 测试 → 部署
     ↓              ↓           ↓            ↓           ↓         ↓        ↓
   2天            2天         5天          3天         5天       4天      2天

关键路径总时长：23天
```

## 协作流程

### 每日协作

1. **晨会**（15分钟）
   - 昨日完成情况
   - 今日计划
   - 阻塞问题

2. **代码/文档提交**
   - 每日至少提交一次
   - 提交信息清晰
   - 关联任务ID

3. **问题升级**
   - 技术问题 → Agent-前端
   - 文案问题 → Agent-策划
   - 进度问题 → PM

### 阶段评审

| 阶段 | 评审内容 | 参与Agent |
|-----|---------|----------|
| 策划评审 | 策划文档 | 全体 |
| 文案评审 | 故事线文案 | 策划、PM |
| 数据评审 | JSON数据 | 数据、前端 |
| 开发评审 | 代码质量 | 前端、测试 |
| 测试评审 | 测试报告 | 全体 |

## 交付物流转

```
策划文档 ──→ 文案创作 ──→ JSON数据 ──→ 前端开发 ──→ 测试验证
    │           │           │           │           │
    ↓           ↓           ↓           ↓           ↓
  评审通过    审核通过    验证通过    自测通过    验收通过
```
