## 第二部分：文案编写规范

### 一、事件编写标准格式

```
事件ID: [角色代码]_[天数]_[时段]_[序号]
事件名称: [简洁有力的标题]
事件类型: [专属/常规/关联]
触发条件: [属性条件/随机触发/前置事件]

事件描述:
[100-200字的情景描述，包含场景、人物、冲突]

选项设计:
选项1: [选项文本]
  - 属性变化: 存款±X, 体重±X, 面子±X, 心情±X, 健康±X, 运气±X
  - 后续影响: [触发事件ID/无]
  - 特殊效果: [获得道具/解锁成就/改变关系]

选项2: [选项文本]
  - 属性变化: ...
  - 后续影响: ...

选项3: [选项文本]
  - 属性变化: ...
  - 后续影响: ...

选项4: [选项文本]（可选）
  - 属性变化: ...
  - 后续影响: ...
```

### 二、选项设计原则

#### 1. 选项数量
- **标准事件**: 2-3个选项
- **重大事件**: 3-4个选项
- **简单事件**: 2个选项

#### 2. 选项类型
| 类型 | 特点 | 适用场景 |
|------|------|----------|
| 安全选项 | 属性变化小，风险低 | 保守型玩家 |
| 冒险选项 | 可能获得高收益或高损失 | 激进型玩家 |
| 角色特色选项 | 符合角色人设 | 所有专属事件 |
| 热梗选项 | 融入网络热梗 | 轻松幽默场景 |
| 隐藏选项 | 需满足条件才显示 | 探索型玩家 |

#### 3. 选项文案要求
- 每个选项15-30字
- 口语化、有画面感
- 避免过于直白，保留悬念
- 融入热梗要自然

**示例**:
- ❌ "给红包" / "不给红包"
- ✅ "慷慨解囊，展现长辈风范" / "装傻充愣，假装没带现金"

### 三、属性变化规则

#### 1. 属性定义
| 属性 | 初始值 | 作用 | 获取/消耗方式 |
|------|--------|------|--------------|
| 存款 | 角色不同 | 购买物品、发红包 | 工作收入、红包收入、消费 |
| 体重 | 角色不同 | 影响健康、外观 | 饮食、运动 |
| 面子 | 50 | 社交评价、触发事件 | 成就、礼物、社交表现 |
| 心情 | 50 | 影响决策质量 | 事件结果、人际关系 |
| 健康 | 50 | 影响活动能力 | 作息、饮食、运动 |
| 运气 | 随机 | 随机事件触发概率 | 随机波动、特殊事件 |

#### 2. 属性变化幅度
| 变化程度 | 数值范围 | 使用场景 |
|----------|----------|----------|
| 微小变化 | ±1-3 | 日常小事 |
| 小幅变化 | ±4-7 | 普通事件 |
| 中等变化 | ±8-12 | 重要决策 |
| 大幅变化 | ±13-20 | 重大事件 |
| 剧烈变化 | ±21+ | 人生转折点 |

#### 3. 属性联动规则
- **存款↓ → 心情↓**: 经济压力影响心情
- **体重↑ → 健康↓**: 过度肥胖影响健康
- **面子↑ → 心情↑**: 社交认可提升心情
- **健康↓ → 运气↓**: 身体不适影响运气
- **心情↓ → 运气↓**: 负面情绪影响运势

### 四、关联事件设计方法

#### 1. 关联类型
| 类型 | 说明 | 示例 |
|------|------|------|
| 前置触发 | 完成A事件才能触发B | 给红包→收到回礼 |
| 属性触发 | 某属性达标触发 | 面子>80触发荣耀事件 |
| 连锁反应 | 选择影响后续多个事件 | 得罪亲戚→被蛐蛐→影响相亲 |
| 隐藏事件 | 特殊条件解锁 | 运气100触发奇遇 |
| 角色互动 | 多角色间事件关联 | A角色选择影响B角色 |

#### 2. 蝴蝶效应设计
```
示例链条:
除夕选择预制菜 → 被长辈吐槽 → 面子-10 → 触发"亲戚蛐蛐"事件 
→ 心情-5 → 影响相亲状态 → 相亲失败 → 触发"父母失望"事件
```

#### 3. 关联事件标注
```
事件A:
  选项1 → 触发事件B（次日）
  选项2 → 触发事件C（3天后）
  选项3 → 属性变化 → 可能影响事件D触发条件
```

### 五、热梗融合指南

#### 1. 融入原则
- **自然融入**: 不生硬插入，符合语境
- **角色适配**: 不同角色使用不同风格热梗
- **时效性**: 优先使用2024-2025年热梗
- **适度原则**: 每个事件1-2个热梗，避免堆砌

#### 2. 融入方式
| 方式 | 示例 |
|------|------|
| 对话融入 | "City不City啊，这身穿搭" |
| 内心独白 | "没事哒没事哒，被问工资而已" |

| 事件标题 | "德华上岗：带娃地狱模式" |
| 结果描述 | "又幸福了姐，收到大红包" |

**Q: 如何处理角色间的互动事件？**
A: 标记为关联事件，在事件描述中说明互动角色，属性变化考虑双向影响。

**Q: 热梗过时了怎么办？**
A: 保留经典热梗（如德华），更新时效性热梗，文档版本迭代更新。

A: 每个角色的专属事件总属性变化应大致平衡，避免某属性过度偏向。
